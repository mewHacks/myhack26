# Hackathon Rules & Judging Criteria — MyHack 2026

## Submission Guidelines

- [ ] AI must be a **key component** of the solution
- [ ] Must address one or more **UN SDGs**
- [ ] All work completed **during the hackathon** (boilerplate/skeletons excepted)
- [ ] Must be **original work** — referencing existing solutions is fine, plagiarism is not
- [ ] Submit **before the deadline** — late/incomplete submissions are not considered
- [ ] Must incorporate **at least one Google Developer technology** as integral to the project

## Content Rules

- No dangerous, vulgar, offensive, indecent, illegal, racist, plagiarized, cruel, or fraudulent content
- Must comply with **laws of Malaysia**
- No material touching **Race, Religion, Royalty (3R)** or **LGBTQ+** topics
- Violations = immediate disqualification; organizers have final say

---

## AI & Tech Stack Requirements

### Mandatory
- AI must meaningfully improve solution effectiveness (functionality, scalability, or impact)
- At least one **Google Developer technology** must be significant and integral

### Google Technologies (examples)
- Google Cloud Platform (GCP) services
- Firebase
- Google Maps API
- Google Workspace
- TensorFlow
- Any other Google-related tools or APIs

### Preferred AI Models
| Model | Strengths |
|-------|-----------|
| Gemini (Google AI) | Search integration, multimodal reasoning (text/image/video/code), real-time info |
| Gemini 3.0 | Advanced reasoning |
| Nano Banana | Lightweight/edge |
| Veo 3.1 | Video generation |
| Gemma | Open-source, fine-tunable |
| Google Labs API models | Experimental capabilities |

> Non-Google AI models are **permitted** but require **strong justification** in the final submission.

---

## Strategic Notes

- Lead with a Google model (Gemini at minimum) to satisfy the spirit of the hackathon
- Use GCP Cloud Run or Firebase for infra to tick Google tech boxes
- Frame the project around an SDG early — judges will look for this
- Prepare a justification if using any non-Google AI (e.g., OpenAI, HuggingFace)
